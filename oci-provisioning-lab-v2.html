ExaCC Provisioning Flow

Cloud@Customer
Key Difference: ExaCC sits in YOUR data center. Oracle manages hardware, but you own the network. Flow: Infrastructure → VM Cluster Network → Validate → VM Cluster → DB Home → Database.
🔴Step-by-Step
1

Create Exadata Infrastructure

Select compartment → Choose ExaCC shape (X8M/X9M) → Configure maintenance window, DNS, NTP, contacts.

☰ → Oracle Database → ExaCC → Exadata Infrastructure → Create
⏱ ~5 min (console) + 4–8 weeks (rack delivery)
2

Download Configuration File → Send to Oracle

Download JSON config. Oracle uses it to provision the physical Exadata rack in your DC.

Infrastructure Details → Download Configuration File (.json)
⏱ 4–8 weeks (physical provisioning)
3

Create VM Cluster Network

Define Client & Backup VLANs with IPs, VLAN IDs, gateway, netmask, DNS, domain. This is internal Exa rack networking, NOT VCN.

Infrastructure Details → VM Cluster Networks → Create
⏱ ~10 min
4

Validate VM Cluster Network

Click Validate → OCI checks connectivity. Status: "Requires Validation" → "Validated".

VM Cluster Network Details → Validate
⏱ ~15–30 min
5

Create VM Cluster

Select validated network → OCPU, memory, storage → SSH keys → License type → GI version.

Infrastructure → VM Clusters → Create VM Cluster
⏱ ~2–4 hours
6

Create DB Home + Database

VM Cluster → DB Home → Oracle version → Create Database: name, PDB, password, workload, backup, TDE.

VM Cluster → DB Homes → Create DB Home → Create Database
⏱ ~45–90 min
7

Connect & Verify

SSH to node via client VLAN IP → Verify CRS, ASM, listener → SQL*Net on port 1521 via SCAN.

ssh -i key.pem opc@<client-ip>
⏱ ~5 min

ExaCS Provisioning Flow

Cloud Service
Key Difference: ExaCS runs in Oracle's OCI DC. Uses standard VCN subnets — no VLANs. Simpler flow: VCN/Subnet → VM Cluster → DB Home → Database.
🟠Step-by-Step
1

Pre-Req: VCN + Subnets + NSGs

Create VCN → Client Subnet (private, /24) → Backup Subnet (private, /24) → NSGs → Service Gateway → Route Table.

☰ → Networking → VCN → Create
⏱ ~15–20 min
2

Create Exadata VM Cluster

Choose shape (X8M/X9M/X10M) → DB servers, OCPU/node, memory → Client + Backup subnet → SSH keys → GI version → License.

⚠ No separate Infrastructure step! Unlike ExaCC, VM Cluster wizard handles infra allocation from Oracle's pool.
☰ → Oracle Database → Exadata on Public Cloud → Create VM Cluster
⏱ ~2–4 hours
3

Create DB Home

Inside VM Cluster → Select Oracle DB version (19c/21c/23ai) → Patch level.

VM Cluster → DB Homes → Create
⏱ ~15–30 min
4

Create Database

DB name, PDB, password, character set → Workload (OLTP/DSS) → Auto backups → TDE → Data Guard (optional).

DB Home → Databases → Create
⏱ ~45–60 min
5

Connect & Verify

OCI Bastion or jump host → SSH to node → Verify SCAN, CRS, ASM → Easy Connect: host:1521/service.

ssh -i key.pem opc@<private-ip>
⏱ ~5 min

ADB-D Provisioning Flow

Autonomous Dedicated
Hierarchy: Exadata Infrastructure → Autonomous VM Cluster → Autonomous Container DB (ACD) → Autonomous Database (ATP/ADW). Port 1522 (not 1521!).
🟢Step-by-Step
1

Pre-Req: VCN + Subnet

VCN → Private client subnet (/24) → Backup subnet → Service GW → NSGs for port 1522.

⏱ ~15 min
2

Create Exadata Infrastructure

Shape → DB + Storage servers → Maintenance schedule → DNS/NTP. Same infra type as ExaCS but consumed by Autonomous VM Clusters.

⏱ ~2–4 hours
3

Create Autonomous VM Cluster

Link to Exa Infra → Client subnet → OCPU + memory → License model. GI auto-managed.

⏱ ~1–2 hours
4

Create Autonomous Container DB (ACD)

Inside AVMC → Patch model (Regular/Autonomous) → Backup dest → Data Guard (optional). ACD is the "shell" for multiple ADBs.

⏱ ~30–60 min
5

Create Autonomous Database

Inside ACD → ATP (OLTP) or ADW (DW) → OCPU + storage → Admin password → Private endpoint → Download wallet for mTLS.

Port 1522 — Download wallet .zip for mTLS. Or use TLS+ACL for wallet-less connections.
⏱ ~10–15 min

DBCS Provisioning Flow

VM/BM
Simplest provisioning. Runs on standard OCI VM/BM shapes, not Exadata. Flow: VCN/Subnet → DB System (includes DB) → Connect.
🔵Step-by-Step
1

Pre-Req: VCN + Subnet

VCN → Single private subnet (no separate backup subnet needed) → NSG with 1521 + 22.

⏱ ~10 min
2

Create DB System

Shape: VM.Standard or BM.DenseIO → Node count (1=single, 2=RAC) → Storage: LVM/ASM → DB version → Subnet → SSH keys → DB created in same wizard: name, PDB, password, workload, backups.

⚠ VM vs BM: VM supports online OCPU/storage scaling. BM gives max performance. RAC needs 2-node.
⏱ ~45–90 min (total)
3

Connect & Verify

SSH → Verify listener, DB status → Easy Connect: private-ip:1521/pdb_name.subnet.vcn.oraclevcn.com

⏱ ~5 min

💻OCI CLI — ExaCS VM Cluster
# Create ExaCS VM Cluster oci db cloud-vm-cluster create \ --compartment-id $COMPARTMENT_ID \ --display-name "vmclus-exacs-lab" \ --subnet-id $CLIENT_SUBNET_ID \ --backup-subnet-id $BACKUP_SUBNET_ID \ --cloud-exadata-infrastructure-id $EXA_INFRA_ID \ --cpu-core-count 4 \ --gi-version "19.0.0.0" \ --hostname "exacslab" \ --ssh-authorized-keys-file ~/.ssh/id_rsa.pub \ --license-model BRING_YOUR_OWN_LICENSE # Create DB Home oci db db-home create \ --vm-cluster-id $VM_CLUSTER_ID \ --db-version "19.22.0.0" \ --display-name "dbhome-19c" # Create Database oci db database create \ --db-home-id $DB_HOME_ID \ --db-name "LABDB" \ --pdb-name "PDB1" \ --admin-password "YourP@ssw0rd#" \ --db-workload OLTP
💻OCI CLI — ADB on Dedicated
# Create Autonomous Database on Dedicated Infra oci db autonomous-database create \ --compartment-id $COMPARTMENT_ID \ --autonomous-container-database-id $ACD_ID \ --display-name "ADB-LAB-ATP" \ --db-name "ATPLAB" \ --cpu-core-count 2 \ --data-storage-size-in-tbs 1 \ --admin-password "YourP@ssw0rd#" \ --db-workload OLTP \ --is-dedicated true # Download Wallet oci db autonomous-database generate-wallet \ --autonomous-database-id $ADB_ID \ --password "WalletP@ss1" \ --file wallet_ATPLAB.zip
